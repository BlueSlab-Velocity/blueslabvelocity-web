# Blue Slab Velocity — website

The production website for Blue Slab Velocity. Astro 5, Tailwind CSS v4, TypeScript strict, deployed to Azure Static Web Apps.

This is a credibility surface, not a CMS-driven marketing site. The performance budget is part of the brief: Lighthouse 100/100/100/100 on a cold deploy, LCP under 1.0 s on throttled 4G, zero JavaScript on routes that do not need it, homepage first-load under 80 KB compressed.

## Stack

- **Astro 5** — static output, View Transitions enabled site-wide, content collections (Zod schemas) for `/writing`
- **Tailwind CSS v4** — CSS-first via `@import "tailwindcss"` and `@theme` in [src/styles/global.css](src/styles/global.css). No `tailwind.config.js`
- **TypeScript** — strict mode, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`, no `any`
- **Self-hosted fonts** — Inter Variable and JetBrains Mono Variable via `@fontsource-variable`, `size-adjust` and metric overrides tuned to the system fallback so font swap causes zero CLS
- **Images** — AVIF with WebP fallback via `astro:assets`
- **Hosting** — Azure Static Web Apps with one HTTP-triggered managed function for the enquiry form
- **CI/CD** — GitHub Actions, deploy on push to `main`

No UI component libraries. No analytics in v1. No third-party scripts.

## Local development

Requirements:

- Node.js (version pinned in [.nvmrc](.nvmrc))
- npm 10+

```bash
nvm use         # picks the version from .nvmrc
npm ci          # clean, lockfile-honouring install
npm run dev     # http://localhost:4321
```

Other scripts:

```bash
npm run build       # production build to ./dist
npm run preview     # serve ./dist locally
npm run check       # astro check (TS + Astro diagnostics)
```

## Authoring `/writing` posts

Posts live in [src/content/writing/](src/content/writing) as MDX files. The filename (without `.mdx`) is the URL slug. Schema is defined in [src/content.config.ts](src/content.config.ts) and validated at build time.

Frontmatter:

```yaml
---
title: "Title in sentence case"
description: "One or two sentences, between 40 and 220 characters."
publishedAt: 2026-04-22
updatedAt: 2026-05-01      # optional
draft: false               # default false; true hides from index, dynamic route, RSS
topics: ["cost", "founders-hub"]
---
```

Body rules:

- Sentence-case headings (`##` and `###`)
- No `#` in the body — the page renders the title as `<h1>` itself
- No bracketed placeholders, no lorem ipsum
- Honour the BSV banned-words list

Reading time is computed from the body. Listing order is by `publishedAt`, descending.

## Deploying to Azure Static Web Apps

This is a one-time setup, then every push to `main` deploys.

### One-time, in the Azure portal

1. Create a Static Web App resource (Standard tier — needed for managed functions and custom CSP headers).
2. Choose **Other** as the deployment source so we can use our own GitHub Actions workflow rather than the auto-generated one.
3. After creation, open **Manage deployment token** and copy the token.

### One-time, in GitHub

1. In the repository, **Settings → Secrets and variables → Actions**.
2. Add a new secret named `AZURE_STATIC_WEB_APPS_API_TOKEN` with the value from the previous step.

### Pipeline

The workflow at [.github/workflows/azure-static-web-apps.yml](.github/workflows/azure-static-web-apps.yml) does the work:

- Checks out, sets up Node, runs `npm ci`
- Sets `PUBLIC_BUILD_ID` and `PUBLIC_COMMIT_SHA` for the footer and colophon
- Runs `npm run build` to produce `./dist`
- Calls `Azure/static-web-apps-deploy@v1` with `app_location: dist`, `api_location: api`, and `skip_app_build: true`

PR builds get a preview URL; the close-PR job tears it down on merge or close.

### Headers and CSP

[staticwebapp.config.json](staticwebapp.config.json) sets:

- A strict Content Security Policy: `default-src 'self'`, no inline script (the theme bootstrap is an external file), no third-party origins
- `Strict-Transport-Security`, `X-Content-Type-Options`, `Referrer-Policy: strict-origin-when-cross-origin`, a minimal `Permissions-Policy`, `X-Frame-Options: DENY`, `Cross-Origin-Opener-Policy: same-origin`
- Long-cache for `/_astro/*` and `/fonts/*`
- A 404 fallback to `/404.html`

If you ever embed Cal.com or another third party, widen `frame-src` and `script-src` in this file. Do it once and audit.

### Domain

In the Azure portal, **Custom domains → Add → Custom domain on Azure DNS or external**. Set `www.blueslabvelocity.com` and a root redirect from the apex.

## Enquiry endpoint

[api/enquiry/index.js](api/enquiry/index.js) is an HTTP-triggered managed function:

- Accepts `POST` only
- Same-origin check via `Origin` and `Referer` (production only)
- Honeypot field (`company_url`) bounces bots silently with a 202
- Validates name, email shape and minimum message length
- Logs to stdout (visible in Application Insights once SWA is wired to it) and redirects to `/contact?sent=1`

There is no email provider in v1. The direct `mailto:hello@blueslabvelocity.com` link in [src/pages/contact.astro](src/pages/contact.astro) is the real channel. To wire SendGrid, Resend or Azure Communication Services, add a sender call in `api/enquiry/index.js` and the relevant outbound domain to the CSP `connect-src`.

## Brand and content guardrails

- Sentence-case headings
- "we" or neutral voice everywhere except `/about`
- Banned words: `leverage`, `synergy`, `ecosystem`, `holistic`, `robust`, `seamless`, `cutting-edge`, `solutions` (standalone), `journey`, `partner with us`, `world-class`, `best-in-class`
- Em dashes only when they earn their keep
- One CTA per page where possible
- Prices in GBP, exclusive of VAT

The package data in [src/lib/packages.ts](src/lib/packages.ts) is the single source of truth. Update prices, scope and deliverables there; the homepage, services index, services detail page and contact form picker all read from it.
